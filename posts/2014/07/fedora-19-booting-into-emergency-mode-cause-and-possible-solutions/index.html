Fedora 19 booting into emergency mode [cause and possible solution(s) ]

Mon 14 July 2014

Booting up my laptop one morning, I found my usual Fedora 19 login screen missing. Instead I found Fedora emergency screen staring at me. It looked something like this (there were few more lines there):

Welcome to emergency mode.
Use "systemctl default" or ^D to activate default mode.
Type “journalctl” to view system logs.
Give root password for maintenance:
(or type Control-D to continue)

As a normal Linux user I tried entering the commands printed on the screen, also looked into the logs for further clue as what exactly went wrong. As I remember I was playing around with the Apache configuration and updated the file systems, day before yesterday. To my dismay any of the commands mentioned in the emergency mode didn't gave any clue.

Searching on the internet gave me some clues that I might need to boot from a live cd/pen-drive to work around. But the problem was that the fedora copy was inside my system. I found many forums which had incomplete solutions for the same.

At last I headed for #fedora channel on IRC and Renich was kind enough to help me with the issue.

The following mentioned steps will also help you understand triggering of emergency mode on Fedora and possibly steps to resolve the same.

What causes an emergency mode to appear

There can be many reasons for triggering of emergency mode such as improper shut down of system, power issues etc. Quoting according to fedora

When things go wrong, there are ways to fix problems.
However, these methods require that you understand the system well.
This chapter describes how to boot into rescue mode, single-user mode,
and emergency mode, where you can use your own knowledge to repair
the system

Oviously I did some nasty things with my system,it might be my system upgradation. But the good news is that you can figure out the real cause of the issue. All you need to do is keep your eyes open. Operating system (here Fedora ) tend to give us reasons for doing anything.

Try pressing 'ctrl+d' to exit and see if there is an error message written. In my case the OS asked me to do manual fsck on the corrupted disk/file system. Something like this:

systemd-fsck[576]: /dev/sda1: UNEXPECTED INCONSISTENCY; RUN fsck MANUALLY.
systemd-fsck[576]: (i.e., without -a or -p options)

The above error message showed me that /dev/sda1 partition needs manual fsck. Another search on the internet gave me how to do fsck in fedora But unfortunatly that didnt solved my problem.

Solution(s) if any???

  1. The first solution can be following the steps written on the screen. One of them being 'ctrl+d'. Try pressing this again and again and you might resolve the issue. I tried this and I did saw the login screen couple of times.

Try rebooting; sometimes, it gets fixed just like that.

If you are still stuck move onto the next step.

  1. As per solution I need a manual fsck on my partition:

    $ fsck.ext4 /dev/sda1

and type y for everything asked. What this did was to repair the filesystem to normal. Oviously I had no idea what was all 'y' for but hey! everything is learning experience, one day I might understand those as well.

  1. Another solution can be typing

    fsck -A

in emergency mode and it might help.

NOTE: Don't just blindly copy paste the commands mentioned above look for the error messages. Your file system might be different as compared to mine. For example I had sda1 and ext4 while you might have ext3 or 2. difference between ext2 ext3 ext4 In my case sda1 partition got a bit screwed and it needed an fsck,look for your issue.

In case steps 2 and 3 work out well for you, try a reboot and hopefully everything will be back to normal.
