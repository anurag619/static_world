how to send multiple pull request on Github

Mon 04 August 2014

Last day I had this situation where I was required to send multiple pull requests to a open source project. All my previous attempts were turning futile as the commits were getting stacked in a single PR (Pull Request). As I searched through Stackoverflow pages I realized I had to create different branches to request multiple PRs. Here are the steps:

  1. You need to create a new branch at first:

    git checkout -b new-branch-name

  2. Now you will have to select a previous commit from where you wish to make changes. This can be from the SHA where the original project was forked:

    git cherry-pick SHA-hash-of-commit

You can get SHA-hash of your commits from:

git log

Alternativly you can even do:

git log --pretty=format:'%h'

to get all SHA-hash but you have to be aware which SHA-hash will work for you.

  1. Make the relative changes and push them (after git add & git commit):

    git push origin new-branch-name

And then you can view the new branch from your Github profile, switch to it and make the pull request.
